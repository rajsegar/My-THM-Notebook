# Database Basics

## Meaning
A database stores information in an organised way so it can be accessed, managed, and updated easily.

## Key Ideas
- A **table** stores data in rows and columns
- A **column** stores one type of data (e.g. name, price)
- A **row** stores one full record (e.g. one order)
- **Relational databases** store structured data in tables
- **Non-relational databases** store data in flexible formats like documents

## Primary Key vs Foreign Key
- **Primary key** = uniquely identifies each row in a table
- **Foreign key** = connects one table to another

## Example Table: Orders

| order_id | drink   | price |
|----------|---------|-------|
| 1        | Coffee  | 3.50  |
| 2        | Tea     | 2.00  |
| 3        | Coffee  | 3.50  |

## Key Point
Think of a database like a spreadsheet — tables are sheets, rows are entries, columns are fields.
